The DMS system applies multiple layers of security and monitoring to protect data and detect anomalies in real-time.

#### 1. IAM — Principle of Least Privilege

Each Lambda function has its **own dedicated IAM role** with only the permissions it needs:

| Lambda | Permissions Granted |
|---|---|
| `documents` | `dynamodb:Query` on Documents table (read-only) |
| `upload-intents` | `dynamodb:PutItem` + `s3:PutObject` on QuarantineBucket only |
| `download-intents` | `dynamodb:GetItem` + `s3:GetObject` on DocumentsBucket only |
| `admin-users` | `cognito-idp:AdminCreateUser`, `AdminAddUserToGroup`, etc. |
| `process-upload` | `s3:GetObject` on Quarantine + `s3:PutObject` on Documents + `dynamodb:PutItem` |

No Lambda function has `*` permissions or cross-service overprivileged access.

#### 2. S3 — Private Buckets & Presigned URLs

- All three S3 buckets have **Block All Public Access** enabled.
- Documents are **never directly accessible** via public URLs.
- Access is granted only through **time-limited Presigned URLs** generated by Lambda:
  - Upload: Presigned PUT URL (expires in ~15 minutes)
  - Download: Presigned GET URL (expires in ~5 minutes)
- S3 server-side encryption (SSE-S3) is applied to all stored objects.

#### 3. GuardDuty — Malware Protection

The `MalwareProtectionPlan` resource scans every file uploaded to QuarantineBucket:

1. File arrives in `quarantine/` prefix via Presigned URL.
2. GuardDuty automatically scans the file.
3. GuardDuty tags the S3 object:
   - `GuardDutyMalwareScanStatus: CLEAN` → safe to process
   - `GuardDutyMalwareScanStatus: THREAT_FOUND` → discard, log to audit
4. Lambda `process-upload` reads the tag before moving the file.

To view scan results:
- Go to **GuardDuty → Malware Protection → Protected buckets**
- Or check the S3 object tags directly in the Console

#### 4. Cognito — Role-Based Access Control

Three user groups enforce different permission levels in the backend:

| Group | Access Level |
|---|---|
| `EMPLOYEE` (precedence 30) | Can upload, view, and download their own documents |
| `DEPARTMENT_ADMIN` (precedence 20) | Can manage documents for their department, share with users |
| `SYSTEM_ADMIN` (precedence 10) | Full access — manage users, view analytics, access all documents |

The group claim is embedded in the JWT token and validated by Lambda at runtime.

#### 5. CloudWatch — Log Monitoring

All 11 Lambda functions write structured logs to CloudWatch Log Groups:
- Log retention: **2 weeks** for dev, **3 months** for production
- X-Ray tracing is enabled on all Lambda functions for end-to-end request tracing

To view logs:
1. Go to **CloudWatch → Log Groups**
2. Find `/aws/lambda/DmsStack-dev-<FunctionName>`
3. Use **Insights** to query: `fields @timestamp, @message | sort @timestamp desc | limit 50`

#### 6. SNS — Cost & Error Alerts

A CloudWatch Alarm is connected to an SNS topic that emails your `alertEmail` when:
- Lambda error rate exceeds a threshold
- Monthly estimated cost goes above budget

To verify alerts are working:
1. Check **SNS → Topics → DmsAlertTopic**
2. Confirm your email subscription is `Confirmed`
3. Check **CloudWatch → Alarms** to see active alarm definitions
